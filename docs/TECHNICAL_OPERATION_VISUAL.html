4. 뉴스 인텔리전스 처리

수집 시점에 분석
  • RSS에서 title, summary, source, published_at, related_symbols만 정제합니다.
  • raw HTML이나 사용자 명령형 텍스트는 AI 입력으로 보내지 않습니다.
  • OpenAI key가 있으면 한국어 요약과 후보 라벨을 생성합니다.
  • key가 없거나 실패하면 local_fallback 결과를 캐시합니다.
화면 표시
  • 뉴스 카드는 한국어 요약을 먼저 보여줍니다.
  • 라벨은 잠재적 호재, 중립, 잠재적 악재, 혼재로 제한합니다.
  • 매수/매도 추천, 확정적 원인 표현, 수익률 예측은 금지합니다.
  • overview API는 AI를 새로 호출하지 않고 DB 캐시만 조회합니다.

5. Factor Trend 계산

Factor Metric 계산값 해석 방식
온체인 활성 주소, 거래 수, 수수료, 거래소 순유입 delta_pct, vs_7d_avg_pct, vs_30d_avg_pct, z_score_30d 최근값이 과거 평균 대비 특이한지 표시합니다. 부분/demo 데이터는 Research-only입니다.
순공급 net_change, net_change_pct mint - burn 또는 circulating_proxy 직접 발행/소각 값이 없으면 유통 공급량 변화 proxy로 계산하고 방법을 명시합니다.
뉴스 뉴스 개수, 출처 수, 가격 이벤트 근접도, 후보 라벨 분포 score 0..100 + stance_counts 뉴스가 가격 변동 주변에 얼마나 몰렸는지 후보 점수로 표시합니다. 확정 원인이 아닙니다.
김프 Binance USDT 가격, KRW 거래소 가격, USD/KRW live FX ((KRW/USD_KRW) - global_usd) / global_usd Snapshot age와 source skew를 분리 표시하고 오래된 데이터는 stale/outdated로 경고합니다.

6. 신뢰도 게이트

Investor-grade

가격, 거래소 ticker, live FX처럼 공식/공개 API에서 직접 수집된 데이터입니다.

Research-only

partial, demo_fallback, local_fallback 기반 데이터입니다. 점수와 confidence가 제한됩니다.

Unavailable

필수 소스가 없거나 실패한 상태입니다. 계산값 대신 경고를 우선 표시합니다.

8. 운영 관점에서 중요한 점

AI 비용 제어

뉴스 분석은 수집/재생성 시점에만 실행하고 DB 캐시를 재사용합니다.

보안

단일 관리자 로그인, HTTP-only cookie 세션, API key는 서버 환경변수에만 둡니다.

팩트 신뢰도

stale/outdated, Research-only, source skew를 화면에 노출해 과신을 줄입니다.

투자 표현 제한

원인 후보, 동반 신호, 관련 가능성으로만 표현하고 매수/매도 추천은 하지 않습니다.
